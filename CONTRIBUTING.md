# 贡献指南

感谢你对「穿越百科全书」的关注！以下是参与贡献的方式。

## 快速开始

1. Fork 本仓库
2. 从 [OUTLINE.md](./OUTLINE.md) 或现有文档中选择一个需要复核、补强或扩写的条目
3. 在对应目录下创建或修改 `.md` 文件，按风险等级选择合适结构
4. 提交 Pull Request

## 条目撰写模板

低风险、可实操条目请包含以下结构（根据内容适当增减）：

```markdown
# 条目名称

> 一句话概述这个技能/知识是什么

## 原理

简要解释背后的科学原理，让读者知其然也知其所以然。

## 所需材料

| 材料 | 获取方式 | 替代方案 |
|------|---------|---------|
| ...  | ...     | ...     |

## 操作步骤

### 步骤 1：xxx
详细说明...

### 步骤 2：xxx
详细说明...

## 常见问题

- **Q**: ...
- **A**: ...

## 安全提醒

如有危险操作，在此醒目标注。

## 参考资料

- ...

## 穿越小说梗

这个技能在穿越小说中通常怎么出现的？现实可行性如何？
```

高风险条目（医药、火药、火器、毒物、强酸强碱、高温冶炼、密闭空间、承重结构、野外采食等）不要套用“材料 + 操作步骤”写成现实教程，应改为：

- 历史背景或原理边界
- 风险识别表
- 不可复现的环节
- 停手信号、撤离距离和事故处置
- 小说可信度判断

## 写作规范

1. **科学准确**：所有内容需有可靠来源，避免传播错误知识
2. **安全第一**：涉及危险操作（火药、火器、强碱、高温、医疗、毒物等）必须写安全边界；会造成现实伤害的步骤、剂量、配方、测试和优化参数不要提供
3. **通俗易懂**：假设读者没有专业背景，解释清楚每一步
4. **古今对照**：标注现代替代方案，让知识在当下也实用
5. **引用来源**：标注参考资料，方便读者深入验证

## 成功标准

一篇条目合格，不只是“读得懂”，而是读者能判断自己是否做对、何时该停手。

| 项目 | 合格要求 |
|------|----------|
| 量化信息 | 高频实操条目写清比例、温度、时间、尺寸、重量或观察窗口 |
| 成败判断 | 有“成功标准”和“失败征兆”，不能只写步骤 |
| 安全边界 | 写清禁忌、停手信号、个人防护和不应复现的危险部分 |
| 来源 | 医疗、化学、食品安全、灾害求生至少核对 2 个可靠来源 |
| 可读性 | 低风险条目让新手能按步骤准备材料、执行、验收和排错；高风险条目让读者能判断风险、停手、隔离和求助 |

## 失败征兆

- 只有故事梗，没有可执行或可判断的知识。
- 只写“适量、少许、加热一会儿”，没有经验尺度。
- 危险条目为了显得厉害，提供威力优化、剂量冒险或民间偏方。
- 医疗和食品条目没有写感染、中毒、腐败、过敏等停手信号。
- 引用来源不可追溯，或只来自短视频、营销号、二手摘要。

## 新增条目提议

如果你想添加大纲中没有的条目：

1. 先在 Issue 中提出，说明：
   - 条目名称
   - 归属哪一卷
   - 为什么值得收录
2. 获得至少一位维护者认可后开始撰写
3. 提交 PR 时关联该 Issue
